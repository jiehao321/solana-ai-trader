# Web3 学习路线图

## 一、基础概念

### 1. 区块链基础
- **区块链**: 分布式账本，不可篡改
- **区块**: 包含多笔交易的数据块
- **共识机制**: PoW (工作量证明), PoS (权益证明)
- **哈希**: 数据的指纹，保证完整性
- **公私钥**: 非对称加密，控制资产

### 2. 以太坊生态
- **EVM**: 以太坊虚拟机，运行智能合约
- **Gas**: 执行操作的手续费
- **Gwei**: Gas 价格单位 (1 Gwei = 10^-9 ETH)
- **Layer 2**: 扩容方案 (Base, Arbitrum, Optimism)

### 3. 代币标准
- **ERC-20**: 同质化代币 (USDC, UNI)
- **ERC-721**: NFT (CryptoPunks, BAYC)
- **ERC-1155**: 多代币标准

## 三、预测市场 (Prediction Markets)

### 什么是预测市场？
让用户对未来事件下注，价格反映概率。

**例子**:
- "Trump 会赢 2024 大选吗？"
- YES 代币价格 = 0.60 → 市场认为 60% 概率

### 为什么有效？
- **群体智慧**: 大量参与者的集体判断
- **真金白银**: 押注让预测更认真
- **实时更新**: 价格随新信息调整

### 主要平台
- **Polymarket**: 最大的加密预测市场
- **Augur**: 去中心化老牌项目
- **Kalshi**: 美国合规预测市场

## 四、交易策略进阶

### 1. 做市商策略详解

**原理**: 同时在买卖两边挂单，赚价差

```
市场: Trump YES
当前价格: 0.50

你的订单:
- 买单: 0.48 (买入 100 份)
- 卖单: 0.52 (卖出 100 份)

如果有人先买你的卖单，再卖给你:
你赚: (0.52 - 0.48) * 100 = $4
```

**风险**: 价格单边运动时，持仓方向错误

### 2. 套利策略

**跨平台套利**:
```
Polymarket: Trump YES = 0.55
Kalshi:     Trump YES = 0.60

在 Polymarket 买入，Kalshi 卖出
利润: 0.05 * 数量
```

**时间套利**:
- 事件前价格被低估/高估
- 事件结果明确后价格收敛到 0 或 1

### 3. 事件驱动策略

**关键时间点**:
- 辩论前建仓
- 数据公布前减仓
- 突发事件后快速反应

## 五、风险管理进阶

### 1. 仓位管理模型

**固定分数法**:
每笔风险 = 账户余额 * 固定比例 (如 2%)

**凯利公式优化**:
```
f = (p * b - q) / b

f = 仓位比例
p = 胜率
q = 败率 = 1 - p
b = 盈亏比
```

### 2. 相关性分析

不要同时押注高度相关的事件:
- ❌ 同时买 Trump 赢 + 共和党赢参议院
- ✅ Trump 赢 + BTC 涨 (相关性低)

### 3. 黑天鹅防护

- 永远保留现金仓位
- 设置最大回撤止损
- 分散到不同类型事件

## 七、智能合约基础

### Solidity 入门
```solidity
// 简单的存储合约
pragma solidity ^0.8.0;

contract SimpleStorage {
    uint256 storedData;
    
    function set(uint256 x) public {
        storedData = x;
    }
    
    function get() public view returns (uint256) {
        return storedData;
    }
}
```

### 关键概念
- **状态变量**: 存储在区块链上的数据
- **函数**: 可执行的操作
- **事件**: 记录重要操作
- **修饰器**: 权限控制 (onlyOwner)

## 八、安全最佳实践

### 钱包安全
- ✅ 硬件钱包存储大额资产
- ✅ 多签钱包管理团队资金
- ✅ 定期备份助记词
- ❌ 不要把私钥存在云端
- ❌ 不要截图助记词

### 交易安全
- ✅ 小额测试后再大额转账
- ✅ 仔细检查合约地址
- ✅ 使用已审计的协议
- ❌ 不要点击不明链接
- ❌ 不要授权给不明合约

### 常见骗局
- 钓鱼网站
- 假空投
- 貔貅盘 (只能买不能卖)
- 私钥/助记词诈骗
